## 三、发现的问题清单

### 🔴 严重问题 (P0)

#### 问题 1: TypeScript 类型安全问题 - 中文属性名
- **文件**: [dashboard/index.vue](file:///Users/speed/Documents/Trae/micro-go/frontend/src/views/dashboard/index.vue#L46-L48)
- **描述**: `coreIndicators` 对象使用了中文属性名 `开户人数`，与 API 接口定义 `accountCount` 不一致
- **影响**: 
  - 类型不安全，容易出错
  - 与 API 接口定义不匹配，需要手动映射
  - 不利于国际化
- **复现步骤**:
  1. 打开 `src/views/dashboard/index.vue`
  2. 查看第 46-48 行和第 169-173 行
  3. 发现 `coreIndicators.开户人数` 与 API 返回的 `accountCount` 需要手动映射
- **预期结果**: 使用英文属性名 `accountCount`，与 API 接口保持一致
- **修复建议**:
  ```typescript
  // 修改前
  const coreIndicators = ref({
    开户人数: { value: 0, change: 0, changeRate: 0 }
  })
  
  // 修改后
  const coreIndicators = ref({
    accountCount: { value: 0, change: 0, changeRate: 0 }
  })
  ```

#### 问题 2: 页面标题未设置
- **文件**: [index.html](file:///Users/speed/Documents/Trae/micro-go/frontend/index.html)
- **描述**: `<title>frontend</title>` 未设置为业务名称
- **影响**: 浏览器标签页显示不专业
- **修复建议**: 改为 `<title>港美股经纪业务数据看板</title>`

### 🟡 中等问题 (P1)

#### 问题 3: 缺少错误边界处理
- **文件**: 多个视图组件
- **描述**: 组件缺少 Vue 错误边界 (errorCaptured) 处理
- **影响**: 组件渲染错误时可能导致整个应用崩溃
- **修复建议**: 添加 `onErrorCaptured` 钩子或使用错误边界组件

#### 问题 4: API 错误处理不够友好
- **文件**: [api/utils.ts](file:///Users/speed/Documents/Trae/micro-go/frontend/src/api/utils.ts#L33-L37)
- **描述**: 响应拦截器只打印 console.error，没有用户友好的错误提示
- **影响**: 用户无法感知 API 错误原因
- **修复建议**: 集成 Element Plus 的 ElMessage 组件显示错误提示

#### 问题 5: 缺少加载骨架屏
- **文件**: 多个视图组件
- **描述**: 使用简单的"加载中..."文本，缺少骨架屏动画
- **影响**: 用户体验不佳
- **修复建议**: 使用 Element Plus 的 `el-skeleton` 组件

#### 问题 6: 数据钻取页面分页逻辑问题
- **文件**: [drilldown/index.vue](file:///Users/speed/Documents/Trae/micro-go/frontend/src/views/drilldown/index.vue#L63-L74)
- **描述**: 分页按钮在数据为空时会报错（访问 null 的属性）
- **影响**: 可能导致运行时错误
- **修复建议**: 添加空值检查

### 🟢 轻微问题 (P2)

#### 问题 7: CSS 变量未定义
- **文件**: [style.css](file:///Users/speed/Documents/Trae/micro-go/frontend/src/style.css#L113)
- **描述**: 使用了 `--text-h`、`--code-bg`、`--accent-bg`、`--border`、`--social-bg`、`--shadow` 等未定义的 CSS 变量
- **影响**: 部分样式可能不生效
- **修复建议**: 在 `:root` 中定义这些变量或删除未使用的样式

#### 问题 8: #app 容器宽度限制
- **文件**: [style.css](file:///Users/speed/Documents/Trae/micro-go/frontend/src/style.css#L211-L221)
- **描述**: `#app { width: 1126px; }` 限制了应用宽度
- **影响**: 在大屏幕上无法充分利用空间
- **修复建议**: 改为 `width: 100%; max-width: 1440px;` 或删除固定宽度

#### 问题 9: 缺少单元测试
- **描述**: 项目未配置任何单元测试框架
- **影响**: 代码质量无法自动化保证
- **修复建议**: 引入 Vitest + Vue Test Utils

#### 问题 10: 缺少 E2E 测试
- **描述**: 项目未配置端到端测试
- **影响**: 用户流程无法自动化验证
- **修复建议**: 引入 Playwright 或 Cypress
